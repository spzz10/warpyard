Drive Warpyard with an AI

Point your AI at Warpyard to build and run servers. Two layers — most tasks use MCP; add an SSH key when your agent needs to configure the server itself.

Manage — any AI client

MCP

Create, resize, power, back up, and point DNS — no keys to paste. Works with Claude, Cursor, or claude.ai in the browser. Set it up below.

Configure — full control

API key + SSH key

An API key drives the REST API; an SSH key lets your agent log into the server and set it up — install software, deploy custom sites, edit configs.

Tip: a local agent like Claude Code can use both at once — MCP to provision, your SSH key to configure. Many sites need neither: push-to-deploy and one-click apps cover the common cases.

Connect an AI client (MCP)

Sign in and approve once — no API key needed. Works with Claude, Cursor, or any MCP client.

Claude Code (CLI)

claude mcp add --transport http warpyard {{ settings.MCP_URL }}/mcp

Then run /mcp in Claude Code, pick warpyard → Authenticate — your browser opens to sign in and approve. Add -s user to make it available in every project.

Claude or Cursor (app)

Add a custom MCP server with this URL:

{{ settings.MCP_URL }}/mcp

Claude: Settings → Connectors → Add custom connector. Cursor: Settings → MCP → Add.

Running the CLI on a remote box? Its login callback goes to a local port, so pin it and forward it from where your browser is:
claude mcp add --callback-port 3118 --transport http warpyard {{ settings.MCP_URL }}/mcp
then ssh -L 3118:localhost:3118 you@that-box before authenticating.
